# Phase Closeout Template

Use this template when closing a roadmap phase. Keep the headings stable so closeout notes stay easy to compare across phases.

## Review Of What Was Built

Summarize the core delivered behavior, docs, or workflow changes.

## Cleanup Review

List what was removed, archived, simplified, or intentionally left deferred.

## Verification Summary

Record the checks that prove the phase is complete. Use the smallest relevant set of checks for the change. Do not add unrelated gates just out of habit.

## Shipped Summary

State what is now true after the phase closes.

## Next Phase

Write the next phase in enough detail that implementation can start without re-inventing the foundation.

## Remaining Roadmap

Add one-line summaries for each later roadmap phase that is still ahead.
